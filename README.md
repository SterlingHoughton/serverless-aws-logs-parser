# serverless-aws-log-parser (WIP)

Make sense of API Gateway and Lambda log files

### Getting started

```bash
npm i --save serverless-aws-log-parser
```

```js
const LogParser = require('serverless-aws-log-parser')

// implement some function get the log file's contents
let logFileContents = readLogFile(/* ... */)

// tell the parser which service the logs are for (api-gateway|lambda)
let parsed = LogParser('api-gateway', logFileContents)
```
### Background

API Gateway + Lambda are the major enablers of serverless architechture nowadays.
While giving developers a pleasant freedom from managing infrastructure, some areas
are still uncovered, calling for third party tools to help developers manage serverless
apps IRL.

One of these areas, which this module is focused on, is understanding the log files
generated by these services.  To say that these files are unreadable would be an
understatement. Use this module to turn the multi-line-per-log with no-clear-delimeter mess of a file into something readable like:

```js
{
  ts_start: '2016-10-01T10:55:45.602Z',
  ts_end: '2016-10-01T10:55:45.697Z',
  method: 'POST',
  request_id: '9ac0eacd-87c5-11e6-b30c-19c3d9849330',
  request_query_string: 'q=koa-oop',
  request_path: 'id=hello',
  resource_path: '/repos/hello',
  response_status: '200',
  duration_ms: 95,
  raw: '<original text>'
}
```

From here, the data should probably be streamed in some way or another into a
data-store in which it can be further, analyzed, monitored etc.
